Key Changes from MySQL

Old (MySQL)

  • Custom user table with password+salt
  • Manual session management
  • ENUM strings in MySQL
  • Raw SQL queries everywhere
  • No foreign key constraints enforced
  • DATETIME columns

New (PostgreSQL + Drizzle)

  • Better Auth manages user/session/account
  • Automatic session handling + JWT
  • PostgreSQL enums (pgEnum)
  • Type-safe Drizzle queries
  • Proper FK constraints with ON DELETE
  • TIMESTAMPTZ columns (timezone-aware)

Better Auth Integration

Better Auth auto-creates user, session, account, and verification tables. We extend the user table with a role field (member/instructor/manager). Profile tables (member, instructor, manager) link to the auth user via userId foreign key.
